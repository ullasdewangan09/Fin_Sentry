# v0.2.0 - 2026-03-10

## Added

- Backend v1 service built with FastAPI
- Data ingestion for vendors, invoices, approvals, and transactions
- Digital twin graph construction using NetworkX
- Rule-based detections for:
  - Segregation of duties
  - Invoice splitting
  - Rapid vendor-to-payment cycles
  - Large payment approval violations
- Investigation endpoint with optional Groq-generated audit narrative

## Changed

- README expanded to include architecture, setup, API flow, and risk rules
- CONTRIBUTING guide expanded with backend development checks
- `.gitignore` refined for Python and local development artifacts

## Fixed

- `.env.example` now uses placeholder value instead of a real-looking key

## Notes

- This release introduces the first backend implementation version.
- Demo dataset is intentionally crafted to trigger all four risk types.
